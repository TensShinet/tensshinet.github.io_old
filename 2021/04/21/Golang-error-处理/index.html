Unwrap 和 Wrap

Wrap 除了带了调用栈上面的信息，还能自定义 msg 信息。在这个库里面还带着有 Wrapf 这样的函数。

使用 fmt.Errorf 的时候如果有 %w，那么会用一个 wrapError 去封装，带有一个 UnWrap 的方法。但不会有栈信息了。

因为有了 Wrap，所以一个错误可以不断封装，那么就有了一个 error chain 的概念。简单来说 error chain 就是封装链。对于封装链来说有三个问题待解决：

  • 如何找到最根本的错误
  • 如何判断这个 err 是不是由某个 err 封装过来的
  • 如何取封装链上的某一个 error

就有了下面三个函数：Cause、Is、As
